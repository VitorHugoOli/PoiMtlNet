## H2 · STL category ceiling, **re-tuned to v17**, all 5 states × {0,1,7,100}  — **the pairing blocker**
> ⚠ **Why this exists (user decision 2026-07-01):** Table 3's Δcat = MTL(v17) − dedicated-STL-cat-ceiling. Today the
> STL cat ceiling exists only at **seed-0 on the v16 recipe** (`../../../docs/results/closing_data/{h100,a40}/*_s0_stl_cat_ceiling.json`).
> An n=20 v17 MTL cell **cannot be paired** against an n=5 v16-recipe ceiling, and leaving the baseline un-tuned while
> the MTL is tuned inflates Δcat (an easy reviewer attack). So re-run the dedicated ceiling with v17's cat knobs, n=20.
- **Recipe** (single-head next_gru → the "per-head cat-lr" is just the cat head at 1e-3; matched to the MTL's cat lever):
  ```bash
  for S in 0 1 7 100; do for ST in alabama arizona florida california texas; do
    python scripts/train.py --task next --state $ST --engine check2hgi_dk_ovl \
        --model next_gru --folds 5 --epochs 50 --seed $S \
        --batch-size 8192 --onecycle-per-head-lr --cat-lr 1e-3 --max-lr 3e-3   # verify flags vs train.py --help
    # score: python scripts/closing_data/score_stl_cat_ceiling.py <rundir> --tag ${ST}_s${S}_stl_cat_ceiling_v17
  done; done
  ```
  Trivial cost (7-class head, ~100 s/run; 20 runs). **A40-offloadable** — if the H100 is busy with H1/H3, run H2 on the
  A40 (see [`A40.md`](A40.md)); it is the fastest GPU item and does not need the big card.
- **Acceptance:** `<state>_s<seed>_stl_cat_ceiling_v17.json` committed; the v17 Δcat re-computed MTL−ceiling on the
  matched folds (pairing discipline, `STATISTICAL_PROTOCOL §4`). **Report the honest (likely smaller) Δcat** — a
  well-tuned baseline is the point.
