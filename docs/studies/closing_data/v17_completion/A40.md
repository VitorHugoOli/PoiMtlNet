## A40-1 · H2 — STL cat ceiling re-tuned to v17  — **fastest; the pairing blocker**
> ⚠ **Why (user decision):** Table 3's Δcat = MTL(v17) − dedicated-STL-cat-ceiling. Today the ceiling exists only at
> **seed-0 on the v16 recipe**, so an n=20 v17 MTL cell **cannot be paired**, and leaving the baseline un-tuned while
> the MTL is tuned inflates Δcat. Re-run the dedicated ceiling with v17's cat knobs, n=20, all states + Istanbul.
- Single-head `next_gru` → the "per-head cat-lr" is just the cat head at 1e-3; matched to the MTL's cat lever. Trivially
  cheap (7-class head, ~100 s/run). **Small states first** (AL/AZ = seconds), then FL, then CA/TX, then Istanbul.
```bash
for S in 0 1 7 100; do for ST in alabama arizona florida california texas istanbul; do
  python scripts/train.py --task next --state $ST --engine check2hgi_dk_ovl \
      --model next_gru --folds 5 --epochs 50 --seed $S \
      --batch-size 8192 --onecycle-per-head-lr --cat-lr 1e-3 --max-lr 3e-3   # verify flags vs train.py --help
  python scripts/closing_data/score_stl_cat_ceiling.py <rundir> --tag ${ST}_s${S}_stl_cat_ceiling_v17
done; done
```
- **Acceptance:** `<state>_s<seed>_stl_cat_ceiling_v17.json` committed; v17 Δcat re-computed MTL−ceiling on the matched
  folds (`STATISTICAL_PROTOCOL §4`). **Report the honest (likely smaller) Δcat** — a well-tuned baseline is the point.
  (Istanbul's ceiling here is on the `dk_ovl` substrate built in H3 — sequence H2-Istanbul after H3's substrate build.)
