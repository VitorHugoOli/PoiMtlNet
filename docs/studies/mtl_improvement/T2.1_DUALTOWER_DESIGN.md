# T2.1 — Reg-private dual-tower: implementation design (proposal, pre-advisor)

**Status:** DRAFT for advisor review (2026-06-04). No code written yet.
**Author:** implementing agent (Tier-2 onboarding, HANDOFF §9).
**Goal:** give the MTL reg head a *full private STAN backbone* on the raw `[B,9,64]`
region sequence, fused with the shared cross-attn pathway, to recover a meaningful
fraction of the MTL→composite reg gap *inside one model*. The thin 1-layer
residual-skip is FALSIFIED (−0.59pp AZ, hard rule 11); this is the un-falsified deeper lever.

---

## 1. What the evidence says the fix must be (recap)

- **P4 (mtl-protocol-fix):** with cat frozen + `cat_weight=0`, MTL reg STILL peaks at ep 2 then
  crashes → the gap is **architectural (pathway), not interference**. STL reg reads region
  embeddings *directly* through its own STAN backbone; MTL reg reads a cat-shaped, joint-trained
  shared-backbone output. Different pathways → different ceilings.
- **§6.4 (B9_STL_STAN_SWAP):** ~75% (~8pp at AZ) of the gap is the **missing private backbone +
  the `next_encoder` 64→256 bottleneck**. Softest sharing (CrossStitch) recovers only ~17%;
  recipe ≤7%; thin skip 0%.
- **T1.3 (DONE):** the upstream encoder is NOT the culprit (prepending `next_encoder` costs ~0pp at
  STL) → the locus is the **joint-training dynamics** (cross-attn co-adaptation / shared-backbone
  handoff). T2.1 runs at full scope (no cheap encoder-bypass shortcut).

**Mechanism the dual-tower restores:** STL reg path = `region_emb[B,9,64] → STAN(64→128→logits)
+ α·log_T`. MTL reg path = `next_input[B,9,64] → next_encoder(64→256) → cross_attn → next_final_ln
→ shared_next[B,9,256] → NextHeadStanFlow(STAN 256→128→logits) + α·log_T`. The dual-tower adds a
**private STAN on the raw 64-dim sequence** (exactly the STL path) and fuses it with the shared
pathway at the reg head.

---

## 2. Architecture

Two new components; minimal blast radius. **Key safety property:** the private tower lives
*inside the reg head* (`self.next_poi`), so it is automatically covered by
`reg_specific_parameters()` and `task_specific_parameters()` (both `yield from
self.next_poi.parameters()`), keeping the param partition bijective+exhaustive with zero edits to
the partition accessors.

### 2.1 New reg head — `NextHeadStanFlowDualTower`
File: `src/models/next/next_stan_flow_dualtower/head.py`; registry name `next_stan_flow_dualtower`.

Constructor params (superset of `NextHeadStanFlow`):
- `embed_dim` (=256, the shared_next dim, injected by `_build_next_head`)
- `raw_embed_dim` (=64, the raw region-emb dim — passed via `--reg-head-param raw_embed_dim=64`)
- `num_classes`, `seq_length=9`, `d_model=128`, `num_heads=4`, `dropout`, `bias_init="alibi"`
- `transition_path`, `alpha_init=0.1`, `freeze_alpha=False`  ← α·log_T prior, **matched to the (a) baseline** (learnable α, prior ON)
- `fusion_mode ∈ {"gated"(b, PRIMARY), "private_only"(a), "aux"(c)}`

Submodules:
- `self.private_stan = NextHeadSTAN(embed_dim=raw_embed_dim, d_model, num_heads, seq_length,
  dropout, bias_init)` — used via `.forward_features()` → `priv_feat [B, d_model]`. This is the
  STL backbone, on raw input.
- `self.shared_stan = NextHeadSTAN(embed_dim=embed_dim, d_model, ...)` → `.forward_features(x)`
  → `shared_feat [B, d_model]`. (This reproduces today's MTL reg backbone.) Built for `gated`/`aux`;
  **not built** for `private_only` (true control).
- Fusion → `feat [B, d_model]`:
  - **gated (b):** `g = sigmoid(gate_net([priv_feat, shared_feat]))` (per-dim, `Linear(2d→d)`);
    `feat = g*priv_feat + (1-g)*shared_feat`. Tests whether the shared backbone *adds* to the
    private tower (the real thesis).
  - **private_only (a):** `feat = priv_feat`. = composite trained jointly (sanity bound, not a
    novel win).
  - **aux (c):** `feat = priv_feat + β·aux_proj(shared_feat)`, `β` a learnable scalar init 0.1
    (≈ private_only at init; can dial in the cross-attn auxiliary if useful).
- `self.classifier = nn.Sequential(LayerNorm(d_model), Dropout(dropout), Linear(d_model, num_classes))`
  applied to `feat`.
- α·log_T prior: **identical** to `NextHeadStanFlow` (read `last_region_idx` from
  `get_current_aux()`; `logits = classifier(feat) + α·log_T[idx]`; same pad/clamp/None-fallback).

Forward: `forward(self, x, raw_region_seq=None)`:
1. `priv_feat = self.private_stan.forward_features(raw_region_seq)`  (raw `[B,9,64]`)
2. if mode≠private_only: `shared_feat = self.shared_stan.forward_features(x)`  (shared_next `[B,9,256]`)
3. fuse → `feat`; `logits = classifier(feat)`; add α·log_T prior.
4. Defensive: `raw_region_seq is None` (isolated unit test / FLOPs probe) → fall back to shared-only
   path (mode-agnostic) so the call never crashes, keeping α in-graph (mirror NextHeadStanFlow's
   None-fallback). Document that this is NOT the training path.

`_uses_raw_region = True` flag so the model knows to pass `raw_region_seq`.

### 2.2 New model — `MTLnetCrossAttnDualTower(MTLnetCrossAttn)`
File: `src/models/mtl/mtlnet_crossattn_dualtower/model.py`; registry `mtlnet_crossattn_dualtower`.

- Override **`forward`** only: identical to parent up to `shared_next`, then
  `out_next = self.next_poi(shared_next, raw_region_seq=next_input)` (raw, post-pad-mask).
  Everything else (encoders, cross-attn stack, cat path, final LNs) inherited unchanged.
- Override **`next_forward`** (isolated reg eval) to also pass `raw_region_seq=next_input`.
- Param partitions (`shared_parameters`, `cat_specific_parameters`, `reg_specific_parameters`,
  `task_specific_parameters`) **inherited unchanged** — private tower ∈ `next_poi` ∈ reg+task. ✓
- Why a subclass and not reuse the `_uses_residual` hook: keeps the falsified thin-skip semantics
  cleanly separate and makes the raw pass-through explicit/auditable. One-method override; tiny.

**Decision point for advisor:** subclass-with-explicit-keyword (proposed) vs reuse the existing
`getattr(self.next_poi,"_uses_residual")` + `residual_input=next_input` plumbing (zero model code).
Proposed = clearer audit trail; reuse = less code. Leaning subclass.

## 4. Gates & protocol (sequential)

1. **Unit-test gate (hard rule 10) — BLOCKS launch.** Synthetic 100-user batch: forward+backward
   shapes OK for all 3 fusion modes; loss finite; param count within ~5% of B9 at D=256;
   `shared / cat_specific / reg_specific_parameters()` **bijective + exhaustive** (assert the
   three sets partition `model.parameters()` exactly, private tower ∈ reg+task); private-tower
   params present in `reg_specific_parameters()`; `setup_per_head_optimizer` builds without error.
2. **Per-arch LR mini-sweep (hard rule 7).** PRIMARY variant (b gated), v14, AL+AZ, 5f×40ep×seed42,
   5 regimes: (R1) constant 1e-3 all-groups; (R2) B9 per-head (cat1e-3/reg3e-3/shared1e-3, cosine
   max-lr 3e-3, alt-opt, α-no-wd, min-best-5); (R3) arch-default = the STL-reg-ceiling recipe for
   the private tower; (R4) per-arch-group LR (private tower its own group at 3e-3 via
   `--reg-head-lr`); (R5) B9 + warmup-5%. Winner → full protocol.
3. **Full protocol (frozen-fold paired, Δ vs frozen (c)/(d)).** At winning regime: variants
   {a,b,c} × {static_weight, PCGrad} × {AL,AZ,FL} × 5f×50ep×seed42. Report {cat,reg}×{disjoint,joint}
   + STL ceiling + composite. Primary axis = reg Acc@10; cat non-inferior (TOST δ=2). Top → 3-seed.
4. **Regime×substrate 2×2 (secondary).** best variant × {v14-fresh, canonical-fresh `gcn_ctrl`} ×
   {B9, dual-tower}. (B9 cells exist; dual-tower cells new; gcn_ctrl substrate exists at FL/AL/AZ.)
5. **HGI sanity probe** if promoted: 2 seeds × AL+AZ × 5f×30ep; escalate if
   |MTL+HGI − STL+HGI| ≥ 2pp.

**Frozen-fold paired:** deterministic `StratifiedGroupKFold(random_state=seed)` → all arms at a seed
share folds automatically. Preflight `freeze_folds.py --check` per state/seed before the sweep.
Score Δ per fold vs the frozen-fold baseline (the (a) v14-B9 run at the same seed), not bare absolutes.

**Decision gate (from the T2.1 card):** shippable win (single-model economy + surviving cat-lift
beats the 2-model composite at ≥2 of {AL,AZ,FL}, cat non-inferior) · scientific partial (>⅓ of the
MTL→composite gap, below composite → report fraction, ship composite) · composite-equivalent (only
private_only wins) · falsified (≈ B9 → irreducibly architectural). All paper-grade.

## 5. Open questions for the advisor

1. **Subclass vs `_uses_residual` reuse** (§2.2) — endorse the explicit-subclass choice?
2. **Fusion granularity:** per-dim sigmoid gate `[B,d]` (proposed) vs scalar gate `[B,1]` vs
   feature-concat→Linear. Per-dim is the most expressive; risk of the gate collapsing to all-private
   (≈ variant a) — is that an acceptable "the data decided" outcome, or should we constrain it?
3. **Where to fuse:** at the pooled `[B,d_model]` feature (proposed) vs at the sequence `[B,9,d]`
   level (fuse before the matching-attention pool). Pooled is simpler and matches "gate(shared_next,
   private_stan(raw))"; sequence-level is more expressive but heavier. Endorse pooled?
4. **α·log_T prior placement:** apply the prior once to the fused `classifier(feat)` (proposed), or
   give each tower its own prior? Single shared prior matches the baseline head; proposed.
5. **`d_model` for the private tower:** the STL ceiling head uses d_model=128 (head default). Keep
   128, or match whatever the frozen STL ceiling used? (Confirm the STL ceiling's d_model so the
   private tower is a faithful STL replica.)
6. **R3 "arch-default" LR regime** definition — use the STL-reg-ceiling training recipe? Confirm.
7. **PCGrad arm:** with `static_weight` the param-partition `zip` in PCGrad isn't exercised; under
   PCGrad the private-tower params must be enumerated. Confirmed they're in `task_specific_parameters()`
   (via next_poi) — any other PCGrad pitfall with a heavier reg head?
8. **Compute scope:** is the {a,b,c}×{2 losses}×{3 states} full grid (18 runs) right, or should the
   PCGrad arm be gated to only the winning fusion mode after the static_weight pass?

## 6. Advisor resolutions (2026-06-04) — applied, GO with revisions

Advisor verdict: **GO**, param-partition auto-coverage claim **confirmed true**, one load-bearing
fidelity fix + several traps. Resolutions adopted:

**Fidelity (P0) — the private + shared towers each faithfully replicate their reference path, via
DISTINCT param names so `_build_next_head`'s injection footgun can't mis-set the backbone:**
- The frozen **(c) STL reg ceiling** recipe (advisor-verified from `scripts/p1_region_head_ablation.py:540`
  + `log.md:363,402`): `NextHeadSTAN` **defaults** — `d_model=128, num_heads=4, dropout=0.3,
  bias_init="alibi", embed_dim=64, seq_length=9`; AdamW **lr=1e-4 wd=0.01**, OneCycleLR max_lr=3e-3
  pct_start=0.3, 50ep bs2048; **α=0 prior-OFF, no tail-loss**. This is the structure the private tower
  must match.
- The **(a) MTL reg backbone** (what `shared_stan` replicates): `next_getnext_hard` built via
  `_build_next_head` → `embed_dim=256, d_model=128, num_heads=8 (INJECTED), dropout=0.1 (INJECTED),
  bias=alibi`, α learnable.
- **Therefore:** head exposes `num_heads`/`dropout` (these get injected = 8 / 0.1 → drive **shared_stan**,
  matching (a)) and **separate `priv_num_heads=4`, `priv_dropout=0.3`** (STL-faithful defaults, NOT
  injected because the names differ → drive **private_stan**, matching (c)). `d_model=128` for both
  (not injected; head default). `bias_init="alibi"` both. This makes: gate→all-shared ⟹ head ≈ (a);
  gate→all-private ⟹ head ≈ (c)-trained-jointly. Clean interpretation.

**Prior ON/OFF (P0):** primary gated arm = **prior ON** (α learnable from 0.1) → apples-to-apples Δ
vs (a). Add a **prior-OFF control** (`alpha_init=0.0 freeze_alpha=True`) on the WINNING fusion mode →
the true (c)-backbone replica + a direct MTL-regime test of O1 (prior helps as KD in MTL, hurts as
additive bias in STL). dropout is decoupled from the prior (structural: priv 0.3, shared 0.1).

**`next_forward` override (P0):** MUST pass `raw_region_seq=next_input` — else isolated reg eval
(the **disjoint diagnostic-best** headline metric) silently runs shared-only and understates the tower.
Unit-test that `next_forward` and `forward(...)[1]` both carry the private-tower contribution.

**Param/recipe traps (P1):**
- `α` stays a direct `self.alpha` attribute of the dual-tower head (NOT nested in a sub-module) so
  `--alpha-no-weight-decay`'s `getattr(model.next_poi,"alpha")` (`helpers.py:156`) resolves. Unit-test
  `isinstance(model.next_poi.alpha, nn.Parameter)` when prior ON.
- None-fallback keeps private tower + learnable `β` (aux mode) in-graph via the `α*0.0` trick
  (`next_stan_flow/head.py:134`) so PCGrad/CAGrad don't crash on `p.grad is None`.
- **wd mismatch (accepted+documented):** STL ceiling used wd=0.01; the MTL recipe +
  `setup_per_head_optimizer` use one global wd=0.05. The integrated model runs the private tower at
  wd=0.05 (part of "can it work inside one model"). R3 LR regime notes this imperfection; add
  `--reg-head-weight-decay` only if results are borderline.
- `raw_region_seq` = the **post-pad-mask** `next_input` (captured after the `masked_fill`), matching
  STL's zero-padded input.

**Unit-test gate additions (P1):** (a) partition bijective+exhaustive AND no `next_poi` param in
`shared_parameters()`; (b) `next_poi.alpha` is `nn.Parameter` when prior ON; (c) param-count gate
loosened to ~10% (two backbones); (d) aux side-channel populated so the prior is actually exercised;
(e) `next_forward` carries the private tower.

**Compute discipline (P2):** PCGrad gated to the winning fusion mode (9 static-weight runs → pick →
3 PCGrad), not the full 18-run cross. Gate-net final-Linear **bias init = +1.0** (sigmoid≈0.73 toward
private) so the under-trained shared backbone must earn its weight; **log mean gate per epoch**.
**Stale-log_T mtime preflight** (`stat` log_T vs next_region.parquet) before every sweep stage.

**Endorsed unchanged:** subclass over `_uses_residual` reuse; per-dim gate unconstrained (collapse =
"data decided"); pooled `[B,d_model]` fusion; single shared prior on fused logits.

**Revised arm plan (sequential):**
1. Unit-test gate.
2. LR mini-sweep: (b) gated, prior-ON, v14, AL+AZ, 5f×40ep×seed42, 5 regimes → winner.
3. Fusion pick: {a,b,c}×prior-ON×static_weight×{AL,AZ,FL}×5f×50ep×seed42 @winner LR → best mode.
4. Winner refinement: best-mode × {prior-OFF control, PCGrad} × {AL,AZ,FL}.
5. Regime×substrate 2×2: best config × gcn_ctrl × {AL,AZ,FL} (B9 cells exist).
6. 3-seed promotion: best config × {0,1,7} × {AL,AZ,FL}.
7. HGI sanity probe if promoted: 2 seeds × AL+AZ × 5f×30ep.
