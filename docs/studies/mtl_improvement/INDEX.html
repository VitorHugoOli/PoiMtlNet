MTL Improvement Track — Experiment Design & Index

An 8-tier, ~32-experiment sequential chain to improve the current B9 MTL recipe on the Check2HGI substrate. Design only as of 2026-05-16 Implementing agent: read §Execution guidelines before writing any code

TL;DR & scope

Goal. Close the −7 to −17 pp next_region gap that MTL currently pays vs matched-head STL across 5 states (paper-canonical v11 §0.1) without losing the small cat lift, by sequentially refining (backbone → loss → batch → optimizer → α → heads) on top of the leak-free Check2HGI substrate.

Out of scope (these have separate tracks; do NOT duplicate):

  • Substrate redesign — ../canonical_improvement/ attacks canonical Check2HGI; ../merge_design/ attacks the merge family.
  • Substrate swap to HGI — ../mtl-exploration/EXPERIMENT_HGI_SUBSTRATE.md established MTL+HGI ≡ STL+HGI under cross-attn. HGI remains a per-arch sanity check here (run once per Tier-2 winner), not a substrate option.
  • Cat-task definition change (single-checkin vs sequence) — 14.4 diagnostic in T0.4 informs a possible follow-up paper; does NOT change the chain.
  • External SOTA baseline reproduction — lives in docs/baselines/.

What this track does: ~30 experiments across 8 tiers attacking the recipe in the order that minimizes re-execution risk. Each tier's winning recipe becomes the next tier's fixed baseline; within a tier, experiments are parallel.

Total compute envelope (revised post-advisor 2026-05-16): ~1700 GPU-h H100 across all tiers (T0 ~80 + T1 ~150 + T2a ~320 + T2b ~450 + T3 ~120 + T4 ~40 + T5 ~100 + T6 ~60 + T7 ~100 + T8 ~300) + ~5–7 weeks engineering. T2b is optional if T2a clears the gate; that path is ~1250 GPU-h.

Scale-conditional or universal recipe?

Decision: the chain explicitly preserves the scale-conditional framing — the final ship recipe in T8 may be B9-prime at large states (FL/CA/TX) and H3-alt-prime at small states (AL/AZ), where “prime” means “after the chain”. Hunting a single universal recipe is permitted as a stretch outcome but is NOT a win condition. This matches the v11 paper framing and prevents a chain that accidentally loses H3-alt's AL/AZ cat advantage chasing FL reg.

Execution guidelines for the implementing agent

Required workflow

  1. Read AGENT_PROMPT.md if present, plus this entire HTML top-to-bottom before touching any code.
  2. Run in a dedicated git worktree on branch mtl-improve branched from main. Do NOT contaminate canonical_improvement or check2hgi-up work.
  3. Use Claude Code's task system (TaskCreate / TaskUpdate). Break down each experiment into unit-test → validate → launch → import → analyze sub-tasks. Mark in_progress before starting; completed only when results are in the experiment's Results placeholder AND a log.md entry is written.
  4. Use the /goal slash command for autonomous experiment runs. Each experiment's Methodology block is a complete goal description. Queue tier-by-tier. When results come in, steer toward the next decision gate.
  5. Respect the chain. Each tier's winning recipe becomes the next tier's baseline. If you ever want to break the chain (run an out-of-order experiment), document the rationale in log.md before launching and assess re-execution risk explicitly.
  6. Mandatory unit-test gate before any new architecture or new head launches multi-fold. Add a test under tests/test_models/test_mtl_improve_variants.py confirming (a) forward+backward shape correctness on a synthetic 100-user batch, (b) loss-finite invariant, (c) parameter count within ~5 % of baseline at D=256, (d) shared_parameters() / cat_specific_parameters() / reg_specific_parameters() partition is bijective and exhaustive (no params orphaned, no params double-counted).
  7. Update this HTML's Results placeholders inline. Per-experiment Results block template lives at the bottom of this section.
  8. Keep log.md current. Append every decision, blocker, falsified hypothesis, direction change. Date every entry absolute (e.g. 2026-05-16, never today).
  9. Mandatory advisor call when (a) a tier completes (b) before declaring the track done. Capture advisor feedback in log.md.
  10. Encouraged: deviate. If results suggest a new path worth exploring — e.g., T2a winner already closes the gap, T2b becomes optional — propose and pursue. Add a new experiment block here and document in log.md.

Hard rules

  • Substrate fixed to Check2HGI — do NOT swap to HGI as the substrate. HGI re-checks are per-T2-winner sanity probes only (T2a.*, T2b.*).
  • Cross-attn and any new task-mixing arch must comply with the existing shared_parameters() / cat_specific_parameters() / reg_specific_parameters() partition so per-head LR continues to work (src/models/mtl/mtlnet_crossattn/model.py lines 378–407 are the contract).
  • Per-fold log_T n_splits=5 contract is mandatory. --folds 1 + per-fold log_T is forbidden (the 2026-05-15 trainer guard hard-fails). Any experiment that mutates fold composition, sequence-stride, or min-history triggers a log_T rebuild via scripts/compute_region_transition.py --per-fold --seed S --n-splits 5 — document the rebuild in log.md as part of the experiment.
  • F51 capacity-widening is FALSIFIED — shared_layer_size > 256 breaks cat at FL. Stay at D=256 for the shared dimension. Capacity sweeps in T2a.3 are depth/head-count only.
  • No fclass-as-feature anywhere. fclass is the linear-probe label; any encoder/loss/head reading fclass is a tautological leak.
  • Statistical claims need n ≥ 10. At n=5 the Wilcoxon floor is p=0.0312. Any “wins paper-canon” claim requires multi-seed (4 seeds × 5 folds = n=20 pooled). For cheap iteration in T2-T7, single-seed n=5 paired tests are acceptable to gate-keep; final shipping (T8) requires multi-seed.
  • Re-run baselines at comparison time. Never quote pre-2026-05-15 numbers from a different machine. T0.7 establishes the same-box canonical baseline.
  • Falsified history (next section) is off-limits. If you believe you have a genuinely different formulation, document the distinction in log.md before launching.

Chain & sequencing

The chain rule: each tier's winning recipe becomes the next tier's fixed baseline. Within a tier, experiments are parallel (run against the same baseline). Diagnostic-only experiments (do not change the recipe) sit in Tier 0 and don't affect the chain.

Why this order minimizes re-execution risk

  1. Tier 0 instrumentation enriches every subsequent experiment's telemetry. Invest once in per-step gradient cosine, α trajectory, F50-D5 saturation curves, MTL-rep generality probes, ECE; every later tier gets the diagnostics for free. Cost is fixed; payoff scales with chain length.
  2. Tier 1 STL refresh pins the comparand. Current Table 2 row 1 in canonical_improvement uses next_gru/next_stan_flow ceilings; T1 produces an updated STL_v2 per-state (multi-seed-light, 3 seeds). MTL deltas in T2–T8 are anchored to a stable, current baseline.
  3. Tier 2 backbone is THE structural change. The F44–F48 disjoint-LR finding, F49 silent-gradient finding, static_weight(0.75) winner, per-head LR, α-no-WD recipe are all cross-attn-conditional. Changing the backbone changes the optimal LR regime, the loss balancer, the gradient interference pattern, and the α dynamics. Backbone goes first; everything downstream is conditioned on it. Critical anti-confound: each candidate gets a per-arch light LR mini-sweep (constant 1e-3 / B9 per-head / arch-suggested-default) so the winner isn't accidentally crowned under a recipe tuned for the loser. See T2a.* for the mini-sweep protocol.
  4. Loss → batch → LR/optim order respects your C6→C4→C7 sequencing. Loss balance comes before batch composition because batch composition's effect partly substitutes for loss-weight effects (oversampling tail classes ≈ class-weighted CE).
  5. α formula and head re-ablation are last because both depend on the consolidated arch+loss+LR+batch recipe. α dynamics depend on shared-LR; head co-adaptation depends on the gradient pattern through the new backbone.

Evaluation framework

Primary metrics

  • Cat macro F1 — on the cat task head, 7-class macro-averaged F1.
  • Reg Acc@10_indist — on the reg task head, in-distribution-restricted Top-10 accuracy. Reported alongside Acc@5 and MRR.
  • Joint Δm (Maninis CVPR 2019 / Vandenhende TPAMI 2021): paired with per-fold STL baselines. Two flavours: PRIMARY = cat F1 + reg MRR, SECONDARY = cat F1 + reg Acc@10.
  • Pareto-front hypervolume across category_weight sweep (T0.5 wiring).

Generality probes (must report for every Tier-3+ winner)

  1. POI fclass linear probe on MTL pre-head shared representation (T0.5 wiring).
  2. POI kNN-Jaccard@10 vs HGI.
  3. Region category-mix R².

Diagnostic probes (every Tier-1+ experiment)

  • Per-step gradient cosine similarity on shared params (sub-agent 7 B1, extend current per-epoch batch-0-only logging).
  • α trajectory per epoch per fold (sub-agent 7 F1).
  • F50-D5 saturation curves per parameter group (cat-encoder, reg-encoder, shared, cat-head, reg-head, α): when does each component stop learning? (sub-agent 7 E1)
  • Confusion matrix per task per fold persisted as confusion_matrix.npy (sub-agent 7 H1).
  • Region-popularity binning of Acc@10 (head/tail decomposition; sub-agent 7 H2).
  • ECE per task per fold (sub-agent 7 L1).
  • Leak-probe value (linear probe on last check-in's MTL pre-head representation, mirror canonical_improvement T1.1 mandatory pre-flight) — reported alongside main metrics for every experiment.

Statistical gates

  • Wilcoxon one-sided strict p<0.05 on the targeted-improvement axis vs the tier's current baseline.
  • TOST non-inferiority, δ=2 pp on the not-targeted axis.
  • Multi-seed n ≥ 20 (4 seeds × 5 folds pooled) for any shipping-grade claim (T8).
  • Single-seed n=5 paired acceptable for Tier-2–7 gate-keeping (Wilcoxon floor p=0.0312).

Mandatory pre-flight

T0.2 (mask/pad propagation) must pass before any Tier-1 launch. Three unit tests under tests/test_models/test_mask_propagation.py assert: (a) pad embeddings stay zero through next_encoder; (b) attention weights to pad positions are zero in cross-attn; (c) heads ignore pad in pooling (or assert documented current behaviour if they don't). If any fails, chain pauses for fix + T1 ceiling re-run.

AL/AZ → FL/CA/TX promotion protocol

Most experiments specify AL+AZ as the first 5-fold run (faster, smaller). Promote to FL+CA+TX only if (a) cat F1 non-inferior at TOST p<0.01 at AL+AZ and (b) the targeted-improvement axis is strictly positive at Wilcoxon p<0.05 at AL+AZ. Always re-run a fresh same-machine canonical baseline at FL/CA/TX at promotion time — never quote pre-existing numbers. The merge_design FL +13 pp stale-baseline incident is the cautionary tale.
